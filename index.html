Behind the Tech: Localized Grid Dispatching Mathematics

1. Spatial Indexing & Complexity Reduction

In global fleet dispatching, we run an O((N+M)3) algorithm. If a city has 5,000 drivers and 2,000 requests, matching takes trillions of operations. Grid Dispatching divides the spatial space into cells Gi,j. Drivers and stations are placed into bucket lists keyed by cell hashes: 

hash(lat, lng) => cell_id(i, j)

For a cell with active station demand, we define a search ring RK of radius K. Matching is limited to:

Dcell ⊆ Gi,j   &   Scandidates ⊆ ∪(u,v)∈RK Gu,v
This localizes the Hungarian algorithm to isolated sub-problems of size ≪ 50, rendering execution instantaneous and highly parallelizable.

2. Localized Batch Bipartite Matching

Unlike Greedy Nearest Neighbor which immediately claims the closest driver (often causing a sub-optimal global state), the Localized Batch algorithm waits for a window of T seconds.

It aggregates all unassigned requests r ∈ Dlocal and available drivers d ∈ Slocal in the search ring. It builds a cost matrix C where:

Cr,d = Distance(r, d) + α · WaitTime(r)
The algorithm runs the Kuhn-Munkres (Hungarian) matching to find the assignment matching set X that:
min ∑r,d Cr,d Xr,d      s.t.  ∑d Xr,d ≤ 1,  ∑r Xr,d ≤ 1
This minimizes total pick-up travel distance and wait time variance.

3. Proactive Grid Rebalancing

High-density commuter stations suffer from rapid driver depletion. Proactive Rebalancing uses historical/scheduled arrival data to generate demand prediction vectors.

If station cell Gi,j has predicted demand D̂i,j higher than current supply Si,j, a rebalancing potential field F is exerted on idle drivers in adjacent cells:

F⃗(Gu,v) = ∑(i,j)∈Ν [ (D̂i,j - Si,j) / Dist((u,v), (i,j))2 ] u⃗
Idle drivers receive a micro-incentive to drift along the vector F⃗ (indicated by orange flow vectors on the map), distributing supply ahead of commuter rushes and achieving a high overall match rate.
